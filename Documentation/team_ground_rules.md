# Team Ground Rules
This document states general rules for the team work.

## General behaviour
The team follows Scrum values to work together.

* __Commitment:__ to achieving goals and supporting teammates.
* __Courage:__ to speak up when necessary.
* __Focus:__ on the work of the Sprint and the Sprint Goal
* __Openness:__ being transparent about work, challenges, and feedback, and being willing to try new things.
* __Respect:__ Valuing each other as capable, independent individuals.

## Meetings
* __Daily Scrum:__ 15 min, at 9:30 every weekday.
* We do calls through Discort
* If somebody is unavailable, that person should communicate that before hand on the teams chat.

## Version Control
* We use Git and GitHub to do version control.
* Features are worked in their own branches
* The branches ar reviewed and merged to 'main' through "Pull Request" (PR).
* The PR needs to be reviewed and approved by at least 2 relevant team members.
* All comments in the review must be resolved before approving.

## Project organization
* We use GitHub Project to track Project Backlog
